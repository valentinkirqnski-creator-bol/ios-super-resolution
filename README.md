# Handheld SR — iOS camera app

A minimal camera app that captures an **8-frame RAW (DNG) burst**, runs the
Handheld Multi-Frame Super-Resolution pipeline (the *same* C++ core as the
Windows/Android builds), and saves one final **×2 (48 MP) image** to your photo
library.

- Big shutter button in the bottom middle; gallery thumbnail to its right.
- **Auto focus + auto exposure driven by the light** (continuous AF/AE), plus
  **tap-to-focus** and a **manual shutter-speed** selector along the top.
- Press the shutter → 8 RAW frames are captured, processed, and saved with a
  live progress bar.

## ⚠️ Important status / honest caveats

This iOS app was written **without access to a Mac, the iOS SDK, or a device**,
so it has **not been compiled, run, or tested**. It is a careful scaffold that
reuses your already-working C++ pipeline. Expect to iterate on real build/run
errors. Known areas most likely to need fixing on first build:

1. **LibRaw compiling for iOS** — we vendor LibRaw source and compile it in the
   app target. Some files/flags may need adjusting for the iOS toolchain.
2. **AVFoundation RAW specifics** — RAW burst capture and `fileDataRepresentation()`
   behavior vary by device; may need tweaks.
3. **Memory** — 8× 12 MP frames + intermediates on a phone is heavy; if it gets
   killed, we reduce the burst size or add tiled loading.

I cannot certify it "crash-free" from here — the cloud build + a couple of test
runs on your phone is how we find and fix the real issues.

## How the build works (no local Xcode, free)

The project is generated by **XcodeGen** from `project.yml`, so there's no
fragile `.xcodeproj` to hand-edit. A free **GitHub Actions macOS runner**
compiles an **unsigned `.ipa`** you can sideload.

### 1. Put the workflow where GitHub can see it
GitHub only runs workflows from the repository root. Copy the workflow up:

```
# from the repo root
mkdir -p .github/workflows
cp ios/.github/workflows/ios-build.yml .github/workflows/ios-build.yml
```

Commit and push to GitHub.

### 2. Run the build
On GitHub: **Actions → “iOS unsigned build” → Run workflow**. When it finishes,
download **`HandheldSRCamera-unsigned-ipa`** from the run's **Artifacts**.

> This uses free GitHub-hosted macOS minutes (free for public repos; private
> repos get a monthly free allotment). Alternatives that also work: **Codemagic**
> (500 free min/month) or **Expo EAS**.

### 3. Install on your iPhone for free (no $99 account)
Use your **free Apple ID** with either tool on your **Windows PC**:

- **AltStore** (recommended): install AltServer on Windows, then use AltStore on
  the phone to install the `.ipa`. It **auto-refreshes** the 7-day signature over
  Wi‑Fi so the app keeps working.
- **Sideloadly**: connect the iPhone via USB, drag in the `.ipa`, sign in with
  your Apple ID, install.

Free-account limits: app signature expires every **7 days** (AltStore
re-signs automatically), max ~3 sideloaded apps. **$0** for personal use.

## Local build (if you do have a Mac)

```bash
cd ios
brew install xcodegen
bash scripts/fetch_libraw.sh
xcodegen generate
open HandheldSRCamera.xcodeproj   # then Run on your device
```

## Project layout

```
ios/
  project.yml                 # XcodeGen spec (sources, flags, Info.plist)
  Sources/
    HandheldSRApp.swift        # @main
    CameraView.swift           # SwiftUI UI (shutter, gallery, shutter picker)
    CameraPreview.swift        # AVCaptureVideoPreviewLayer + tap-to-focus
    CameraModel.swift          # capture session, 8x RAW burst, AF/AE, save
    Bridge/
      SRBridge.h / .mm         # Obj-C++ -> C++ core
      HandheldSR-Bridging-Header.h
    Info.plist
  scripts/fetch_libraw.sh      # vendors LibRaw source
  .github/workflows/ios-build.yml
```

The heavy lifting (`grey_pyramid/align/robustness/kernels/merge/pipeline/
raw_io/dng_writer`) is the shared core under
`../android/app/src/main/cpp/core` — the exact code validated on the desktop.
